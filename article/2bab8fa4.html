Redis6.0+版本修改密码无效

1、问题描述
在搭建redis主从集群的时候，一直出现密码问题，之前设置的是默认密码123456.后来发现即使修改了默认密码之后密码还是需要写原来的才能登入，甚至注释requirepass后，还是需要123456的密码。
看了很多文章，要么没法解决，要么下次登入后还是一样的问题，不是一次性能解决的
2、解决方案：

修改redis.conf 中的 user default on nopass ~* &* +@all
注意默认改成这个nopass！！！
没改之前是一长串加密的东西，我猜就是123456，改成nopass，或者改成自己特定的密码，或者直接删除generated by config rewrite 下面这些东西，问题解决！

3、成功无密码登入
先看一下线程 ps aux | grep redis ,查出号码后kill，重启一下redis（也重启下xshell），大功告成！！
